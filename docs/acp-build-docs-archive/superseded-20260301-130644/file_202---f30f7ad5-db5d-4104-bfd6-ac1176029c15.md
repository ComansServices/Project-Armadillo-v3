## Example: Filled Template

### Task: [INFRA-07] — MinIO Storage Setup

**Sprint:** 0
**Priority:** P0
**Effort:** 1d
**Agent:** infra-engineer
**Branch:** `sprint-0/INFRA-07-minio-storage`
**Status:** new

### What Exists

- `docker-compose.yml` has 5 services (postgres, redis, api, worker, web). No MinIO.
- No object storage client anywhere in the codebase.
- `.env.example` exists but has no S3 vars.

### What to Build

1. Add `minio` service to `docker-compose.yml` (image: `minio/minio:latest`, ports 9000/9001, volumes, env vars)
2. Add `minio-data` named volume
3. Install `@aws-sdk/client-s3` in apps/api
4. Create `apps/api/src/lib/s3.ts` with S3Client configured for MinIO
5. Add bucket auto-creation on startup: `armadillo-evidence`, `armadillo-reports`
6. Add S3 env vars to `apps/api/src/config.ts` (Zod validation)
7. Write smoke test: upload test file, download, verify content matches

### Acceptance Criteria

- [ ] `make up` starts MinIO alongside other services
- [ ] MinIO console accessible at http://localhost:9001
- [ ] S3 client can upload and download a test file
- [ ] Buckets created automatically on first start
- [ ] Works on both amd64 and arm64
- [ ] `pnpm build` passes

### Dependencies

| Dep | Status | What It Provides |
|---|---|---|
| None | — | No dependencies |

### Files to Touch

- `docker-compose.yml` — add minio service + volume
- `apps/api/package.json` — add @aws-sdk/client-s3
- `apps/api/src/lib/s3.ts` — (new) S3 client + bucket init
- `apps/api/src/config.ts` — add S3_ENDPOINT, S3_ACCESS_KEY, S3_SECRET_KEY
- `.env.example` — add S3 vars

### Commit

```bash
git add -A
git commit -m "[INFRA-07] add MinIO container and S3 client for evidence storage"
git push origin HEAD
```
