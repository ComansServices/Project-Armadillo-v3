# TESTING.md — Test Strategy & Validation Process

> How we test. Patterns to follow. Validation checklists.

---

## Testing Layers

| Layer | Tool | Scope | When |
|---|---|---|---|
| **Smoke tests** | `make test` (shell scripts) | API health, queue connectivity, basic endpoints | Every commit |
| **Type checking** | `pnpm build` (TypeScript strict) | All packages compile with no errors | Every commit |
| **E2E tests** | Playwright | Happy-path user flows across UI + API | Every sprint |
| **Visual testing** | OpenClaw (primary) / Agent Browser (backup) | Responsive design, visual regressions | Sprint validation |
| **Manual testing** | Browser + curl | Edge cases, exploratory testing | Sprint validation |
| **Integration tests** | Custom test scripts | Cross-tenant isolation, RLS verification | After INFRA-01/02 |

---

## Pre-Commit Checklist (MANDATORY)

Before every `git commit`:

```bash
# 1. TypeScript compiles
pnpm build

# 2. Smoke tests pass
make test

# 3. No console.log left in production code
grep -r "console.log" apps/api/src/ apps/worker/src/ --include="*.ts" | grep -v "// debug"

# 4. No `any` types
grep -r ": any" apps/api/src/ apps/worker/src/ packages/*/src/ --include="*.ts"
```

All four must pass. If `pnpm build` or `make test` fails, do NOT commit.

## Playwright E2E Patterns

### File Structure

```
e2e/
├── phase7.spec.ts          # Existing tests (remediation, search, badges)
├── sprint-0.spec.ts        # Sprint 0 foundation tests
├── sprint-1.spec.ts        # Sprint 1 intelligence tests
├── sprint-{N}.spec.ts      # One file per sprint
├── fixtures/
│   └── test-data.ts        # Shared test data
└── helpers/
    └── auth.ts             # Auth helpers for tests
```

### Test Template

```typescript
import { test, expect } from '@playwright/test';

test.describe('Feature Name', () => {
  test.beforeEach(async ({ page }) => {
    // Set dev auth headers
    await page.setExtraHTTPHeaders({
      'x-armadillo-user': 'admin@test.com',
      'x-armadillo-role': 'owner',
    });
  });

  test('should do the thing', async ({ page }) => {
    await page.goto('http://localhost:3000/vulns');
    
    // Wait for data to load
    await expect(page.locator('table tbody tr')).toHaveCount.greaterThan(0);
    
    // Assert
    await expect(page.locator('[data-testid="vuln-list"]')).toBeVisible();
  });
});
```

### Common Patterns

**Navigate and wait for API:**
```typescript
await page.goto('http://localhost:3000/vulns');
await page.waitForResponse(resp => resp.url().includes('/api/v1/vulns') && resp.status() === 200);
```

**Test API directly:**
```typescript
const response = await page.request.get('http://localhost:4000/api/v1/vulns', {
  headers: {
    'x-armadillo-user': 'admin@test.com',
    'x-armadillo-role': 'owner',
  },
});
expect(response.status()).toBe(200);
const body = await response.json();
expect(body.items.length).toBeGreaterThan(0);
```

**Test form submission:**
```typescript
await page.fill('[data-testid="assignee-input"]', 'jane@example.com');
await page.click('[data-testid="save-button"]');
await expect(page.locator('[data-testid="success-toast"]')).toBeVisible();
```

**Test responsive (mobile):**
```typescript
test('mobile layout', async ({ page }) => {
  await page.setViewportSize({ width: 375, height: 812 });
  await page.goto('http://localhost:3000/');
  await expect(page.locator('[data-testid="mobile-nav"]')).toBeVisible();
});
```

## Sprint Validation Process

After completing all tasks in a sprint:

### 1. Run All Tests

```bash
# Type check
pnpm build

# Smoke tests
make test

# Playwright e2e
pnpm exec playwright test

# View report
pnpm exec playwright show-report
```

### 2. Visual Verification

Use OpenClaw browser control (primary) or Agent Browser (backup):

- [ ] Navigate all new pages — load correctly, no errors
- [ ] Test at 1440px (desktop) and 375px (mobile)
- [ ] Verify dark mode (after INFRA-10/US-OPS.04)
- [ ] Test keyboard navigation (Cmd+K, Tab, Enter)
- [ ] Check all new API endpoints via curl

### 3. Write Validation Report

Create `docs/sprint-{N}-validation.md`:

```markdown
# Sprint {N} Validation Report

**Date:** YYYY-MM-DD
**Agent:** [who ran validation]
**Branch:** sprint-{N}/main

## Task Completion

| Task | Status | Notes |
|---|---|---|
| STORY-ID | ✅ Pass | Working as specified |
| STORY-ID | ⚠️ Partial | [describe what's missing] |
| STORY-ID | ❌ Fail | [describe failure] |

## Tests Run

| Suite | Result | Duration |
|---|---|---|
| pnpm build | ✅ Pass | 12s |
| make test | ✅ Pass | 3s |
| Playwright | ✅ 15/15 Pass | 45s |

## Issues Found

1. [Issue description + severity + recommended fix]
2. None

## Sign-Off

- [ ] All P0 tasks complete
- [ ] All tests pass
- [ ] No TypeScript errors
- [ ] Demo dataset updated (if needed)

```

## Cross-Tenant Isolation Tests (Critical)

After INFRA-01 + INFRA-02, these tests MUST pass:

```typescript
test.describe('Multi-tenant isolation', () => {
  test('User A cannot read Project B vulns via API', async ({ request }) => {
    // User A has access to Project A only
    const response = await request.get('/api/v1/vulns', {
      headers: {
        'x-armadillo-user': 'userA@test.com',
        'x-armadillo-role': 'staff',
        'x-armadillo-project': 'project-a-id',
      },
    });
    const body = await response.json();
    
    // Verify NO vulns from Project B appear
    for (const vuln of body.items) {
      expect(vuln.projectId).toBe('project-a-id');
    }
  });

  test('Direct Prisma query respects RLS', async () => {
    // Even with raw query, RLS prevents cross-tenant access
    // This test runs in the API context with project context set
  });

  test('API without project context returns empty', async ({ request }) => {
    // Request without project scope should return no tenant data
  });
});
```

## Definition of Done (Per Feature)

Every feature is "done" when ALL of the following are true:

- [ ] Code merged via reviewed PR
- [ ] Unit tests: 80%+ coverage for new code
- [ ] Playwright e2e test for happy path
- [ ] API documented with request/response examples
- [ ] Mobile-responsive (375px width minimum)
- [ ] No TypeScript strict mode errors (`pnpm build` passes)
- [ ] CI passes (GitHub Actions)
- [ ] Demo dataset updated if new models added
- [ ] All API request/response bodies validated with Zod
- [ ] All tenant-scoped tables have RLS policies
- [ ] External API responses parsed with Zod `.safeParse()`

## CI/CD Pipeline

GitHub Actions: `.github/workflows/phase4-gate.yml`

```yaml
# Triggers on push to main and PR
jobs:
  gate:
    steps:
      - pnpm install
      - pnpm build          # TypeScript strict
      - make test            # Smoke tests
      - pnpm exec playwright test  # E2E (when configured)
```

All checks must pass before merge to main.
