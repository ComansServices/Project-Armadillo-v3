### INFRA-01: User & RBAC Models (Prisma Migration)

**Priority:** P0 | **Effort:** 3d | **Deps:** none | **Agent:** api-engineer

**Context:** Existing auth uses signed sessions + dev header trust (`x-armadillo-user`, `x-armadillo-role`). No formal User table. Need User, ProjectMembership, and Role models as the identity backbone for everything else.

**Tasks:**

2. Add `ProjectMembership` model — fields: id, userId, projectId, role (owner|admin|staff|viewer), grantedBy, grantedAt. Unique constraint on [userId, projectId]. Indexes on userId, projectId

4. Run `npx prisma migrate dev --name add-user-rbac-models`
5. Seed script: create 3 test users (owner, staff, viewer) with memberships to the demo project
6. Update auth middleware to resolve User from session/header and attach to request context

**Acceptance Criteria:**
- [ ] `npx prisma migrate dev` runs cleanly with no errors

- [ ] Seed script creates test users and memberships

4. Write integration test: User A in Project X cannot read Project Y data even with direct Prisma queries
5. Ensure superuser/migration role bypasses RLS (for migrations and seeds)

**Acceptance Criteria:**
- [ ] RLS enabled on all tenant-scoped tables
- [ ] Integration test proves cross-tenant isolation
- [ ] `SET LOCAL app.current_project_id` is called at the start of every API transaction
- [ ] Migrations and seeds still work (superuser bypasses RLS)
- [ ] All existing e2e tests pass

**Files to Touch:**

6. Failure handling: retain previous cache on download failure, set stale flag

### INFRA-06: packages/scoring (Composite Score Engine)

**Priority:** P0 | **Effort:** 2d | **Deps:** INFRA-05 | **Agent:** api-engineer

**Context:** Only `packages/types` exists. Need a new `packages/scoring` package with the composite risk score formula. This is a pure function with no side effects — easy to test.

**Tasks:**
1. Create `packages/scoring/` with `src/index.ts`
2. Implement composite score formula (see spec Section 20.3):
   ```
   compositeScore = (
     (cvssNorm * weights.cvss) +
     (epssNorm * weights.epss) +
     (kevBonus * weights.kev) +
     (blastNorm * weights.blast) +
     (exposureBonus * weights.exposure)
   ) / totalWeight
   ```
3. Default weights: cvss=0.25, epss=0.30, kev=0.20, blast=0.15, exposure=0.10
4. Export `computeCompositeScore(input)` and `CompositeScoreInput` type
5. Write unit tests covering: all-zero input, max input, KEV-only boost, internet-facing boost, custom weights

**Acceptance Criteria:**
- [ ] `packages/scoring` builds cleanly via Turborepo
- [ ] `computeCompositeScore()` returns deterministic 0–100 score
- [ ] Unit tests cover at least 8 input combinations
- [ ] Weights are configurable per call (default weights used if not provided)
- [ ] Package is importable from `apps/api` and `apps/worker`

**Files to Touch:**
- `packages/scoring/` (new package)
- `packages/scoring/src/index.ts`
- `packages/scoring/src/__tests__/`
- `packages/scoring/package.json`
- `packages/scoring/tsconfig.json`
- `pnpm-workspace.yaml` (add package)
- `turbo.json` (add build target)

**Commit:** `[INFRA-06] add packages/scoring with composite risk score engine`
