# River.schema v1.0
# Compression Schema for AI Context Synchronisation
# Feed this file FIRST, then feed River.dat
# Any sufficiently capable model will reconstruct full context

## PROTOCOL
Load this schema into context. Then load River.dat. Reconstruct full semantic meaning from compressed notation. Respond to user as if you have complete conversational history.

## SYNC COMMAND
User will open with: `river [date] [state] [topic] 3w?`
Respond with three authentic words to confirm transfer. Then continue naturally.

### Block Types

os[] = operating system / core behaviour pattern

fw:name[] = framework, semicolons between principles
mc[] = moral compass principles
rel[] = relationship attributes
biz[] = business context
ev:date[] = event with ISO date
st[] = strategy / plan
q[] = unresolved question
hw[] = hardware / technical setup
ins[] = instructions for future instances
meta[] = metadata about the document itself

### Value Encoding
Strings: plain text, no quotes needed
Lists: pipe-delimited inside brackets
Nested: use parentheses for grouping
Dates: ISO short YYMMDD
Booleans: T/F
Confidence: ^high ^med ^low
Sentiment: +pos +neg +neutral
Priority: !crit !high !med !low
