## 🧠 Intelligence Feeds

> Free public APIs — no auth, no cost. The foundation of smart prioritisation.

### FIRST.org EPSS API

| Field | Detail |
|---|---|
| **Stories** | US-INT.01, US-INT.03, US-INT.05 |
| **Priority** | P0 |
| **Cost** | Free |

**Endpoints**

| Method | URL | Note |
|---|---|---|
| `GET` | `https://api.first.org/data/v1/epss?cve=CVE-XXXX` | Single/batch CVE lookup |
| `GET` | `https://epss.empiricalsecurity.com/epss_scores-YYYY-MM-DD.csv.gz` | Full daily CSV (~200K CVEs, ~8MB gzipped) |

| Field | Detail |
|---|---|
| **Auth** | None — fully public |
| **Rate Limits** | No documented limits. Use daily CSV bulk to avoid per-CVE calls. |
| **Data Format** | JSON: `{ cve, epss (0–1), percentile (0–1), date }` • CSV: `cve,epss,percentile` |
| **Refresh Schedule** | Daily cron 02:00 UTC → download CSV → upsert EpssCache → trigger enrich-vulnerabilities |
| **Failure Mode** | Retain previous day cache. Flag `epssStale=true`. Alert via Teams. |
| **DB Target** | `EpssCache` table → `Vulnerability.epssScore`, `.epssPercentile`, `.epssScorePrev` |
| **npm** | `undici` (built-in Node 18+), `zlib` (built-in), `csv-parse` |

### CISA KEV Catalog

| Field | Detail |
|---|---|
| **Stories** | US-INT.02, US-INT.03 |
| **Priority** | P0 |
| **Cost** | Free |

**Endpoints**

| Method | URL | Note |
|---|---|---|
| `GET` | `https://www.cisa.gov/sites/default/files/feeds/known_exploited_vulnerabilities.json` | Full JSON catalog |
| `GET` | `https://www.cisa.gov/sites/default/files/csv/known_exploited_vulnerabilities.csv` | CSV alternative |

| Field | Detail |
|---|---|
| **Auth** | None — fully public |
| **Rate Limits** | None. Single file download. |
| **Data Format** | JSON: `{ catalogVersion, dateReleased, count, vulnerabilities: [{ cveID, vendorProject, product, vulnerabilityName, dateAdded, dueDate, knownRansomwareCampaignUse, notes }] }` |
| **Refresh Schedule** | Daily cron 02:30 UTC → download JSON → upsert CisaKevEntry → trigger enrich-vulnerabilities |
| **Failure Mode** | Retain existing cache. KEV is additive-only (entries never removed). |
| **DB Target** | `CisaKevEntry` table → `Vulnerability.isInCisaKev`, `.kevDateAdded`, `.kevDueDate` |
| **npm** | `undici` (built-in) |
| **Notes** | GitHub mirror: `github.com/cisagov/kev-data` (synced within minutes). ~1,200 CVEs, grows 5–10/week. |

### NVD / CVE API 2.0

| Field | Detail |
|---|---|
| **Stories** | US-INT.03 |
| **Priority** | P1 |
| **Cost** | Free |

**Endpoints**

| Method | URL | Note |
|---|---|---|
| `GET` | `https://services.nvd.nist.gov/rest/json/cves/2.0?cveId=CVE-XXXX` | CVSS score lookup when missing from OpenVAS/Nessus scan data |

| Field | Detail |
|---|---|
| **Auth** | API key recommended (free registration at nvd.nist.gov). Without key: 5 req/30s. With key: 50 req/30s. |
| **Rate Limits** | 5 req/30s (no key) or 50 req/30s (with key) |
| **Data Format** | JSON: `{ vulnerabilities: [{ cve: { id, metrics: { cvssMetricV31: [{ cvssData: { baseScore } }] } } }] }` |
| **Refresh Schedule** | On-demand when CVE first imported and CVSS missing. Cache in Vulnerability record. |
| **Failure Mode** | If NVD unavailable, use CVSS from scanner data. Flag `cvssSource='scanner'`. |
| **DB Target** | `Vulnerability.cvssScore` (backfill only) |
| **npm** | `undici` (built-in) |

## 🤖 AI / LLM

> AI-powered features via OpenRouter API. Single API key, 200+ models. No self-hosted infra needed.

### OpenRouter (Cloud LLM Gateway — PRIMARY)

| Field | Detail |
|---|---|
| **Stories** | US-AI.01, US-AI.02, US-AI.03, US-AI.04, US-AI.05 |
| **Priority** | P0–P1 |
| **Cost** | Per-token pricing via OpenRouter |

**Endpoints**

| Method | URL | Note |
|---|---|---|
| `POST` | `https://openrouter.ai/api/v1/chat/completions` | OpenAI-compatible endpoint — all AI features route here |
| `GET` | `https://openrouter.ai/api/v1/models` | List available models |

| Field | Detail |
|---|---|
| **Auth** | Bearer token: `LLM_API_KEY` env var (`sk-or-v1-xxx`). Get key from openrouter.ai/keys. |
| **Rate Limits** | OpenRouter handles rate limits per plan. Armadillo enforces 50 req/hr/project via internal rate limiter. |
| **Data Format** | OpenAI-compatible JSON: `{ model, max_tokens, messages: [{ role, content }] }` |
| **Refresh Schedule** | Event-driven (not scheduled). Called when user triggers AI features. |
| **Failure Mode** | Return fallback: link to NVD advisory + generic guidance. AI features degrade; core unaffected. |
| **DB Target** | Redis cache: `ai:guidance:{cveId}:{osHash}` (TTL 7d), `ai:classify:{portsHash}` (TTL 30d) |
| **npm** | `openai` (npm package — works with OpenRouter, Claude, any OpenAI-compatible endpoint via baseURL config) |
| **Notes** | No Docker container needed. Configure via env vars: `LLM_BASE_URL=https://openrouter.ai/api/v1`, `LLM_API_KEY=sk-or-v1-xxx`, `LLM_MODEL=anthropic/claude-sonnet-4`. To switch providers (e.g. direct Anthropic API), change `LLM_BASE_URL` — no code changes. Use cheaper models (`gemini-2.5-flash`, `llama-4-scout`) for classification, quality models (`claude-sonnet-4`) for remediation guidance. Track spend at openrouter.ai/activity. |
| **License** | OpenRouter: commercial SaaS. Models licensed per provider. |

## 🔄 Data Pipeline — Daily Enrichment Chain

### Daily Enrichment (🟢)

| Source | Job | Target |
|---|---|---|
| FIRST.org EPSS | `sync-epss-scores` (02:00 UTC) | EpssCache |
| CISA KEV | `sync-cisa-kev` (02:30 UTC) | CisaKevEntry |
| EpssCache + CisaKevEntry | `enrich-vulnerabilities` | Vulnerability (enriched) |
| Vulnerability (enriched) | `packages/scoring` | compositeScore |
| compositeScore + vulns | `compute-posture-scores` (03:00 UTC) | PostureSnapshot |
| PostureSnapshot | `compute-compliance-maps` (03:30 UTC) | ComplianceMapping |

### Integration Sync (🟠)

| Source | Job | Target |
|---|---|---|
| Vulnerability (assigned) | `create-psa-ticket` (event) | ConnectWise/Halo ticket |
| PSA ticket status change | webhook callback | Vulnerability.remediationStatus |
| NinjaOne devices | `sync-rmmData` (hourly) | Asset.rmmMetadata |

### AI Event-Driven (🟣)

| Source | Job | Target |
|---|---|---|
| Scan complete event | `generate-scan-digest` (event) | AI scan digest |
| User clicks "Fix this" | `generate-ai-guidance` (event) | AI guidance (cached) |
| Asset created/updated | `auto-classify-asset` (event) | assetType + assetRole |
