### Value Encoding
Strings: plain text, no quotes needed
Lists: pipe-delimited inside brackets
Nested: use parentheses for grouping
Dates: ISO short YYMMDD
Booleans: T/F
Confidence: ^high ^med ^low
Sentiment: +pos +neg +neutral
Priority: !crit !high !med !low
Attribution: attr=@prefix (which model contributed this entry)
Dependency: dep=block:label (this entry depends on another entry)

### Evolution Tracking
Mark entries as superseded when thinking evolves. Old entries compress harder over time but preserve the reasoning chain.
Status tags (append inside brackets):
   ^current = active/latest version of this idea
   ^old(YYMMDD) = superseded on date, kept for context
   ^evolved(label) = pointer to the entry that replaced this one
   ^from(label) = pointer to the entry this evolved from
   ^resolved(YYMMDD, summary) = question answered, preserves the thread
   ^dead(reason) = tried and failed, do not revisit
Usage:
   - Only tag genuine paradigm shifts, not minor refinements
   - Old entries are candidates for aggressive temporal decay (L3)
   - Models should treat ^old entries as historical context, not active guidance
   - Models should never suggest approaches tagged ^dead
   - The evolution chain (^from/^evolved) lets models reconstruct how thinking changed
Example:
   @fw:example[old-belief-here|^old(260217)|^evolved(fw:example-v2)]
   @fw:example-v2[new-belief-here|^current|^from(fw:example)]
