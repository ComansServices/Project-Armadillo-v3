# API_CONTRACTS.md — API Endpoint Reference

> All endpoints with request/response shapes. Read for any API work.

## Authentication

### Dev Mode (Header Trust)
```
x-armadillo-user: admin@test.com
x-armadillo-role: owner
x-armadillo-auth: dev-session-token
```
Disabled in production via `AUTH_ALLOW_LEGACY_HEADERS=false`.

### Production (Signed Session)
Encrypted session cookie set via `/auth/login`.

### API Key (Integrations)
```
Authorization: Bearer armadillo_xxxxxxxxxxxxxxxx
```

## Existing Endpoints (40+)

### Scans

**POST /scans** — Queue a new scan
```json
// Request
{ "targets": [{"type": "cidr", "value": "192.168.1.0/24"}], "config": {} }
// Response: 201
{ "id": "cuid", "status": "queued", "targets": [...] }
```

**GET /scans** — List scans (paginated)
```
?page=1&limit=50&status=completed&projectId=xxx
```

**GET /scans/:scanId** — Scan detail

**GET /scans/:scanId/events** — Event log for scan

**GET /scans/attention** — Failed scans needing attention
```json
// Response
{ "items": [...], "trend": { "7d": [...sparkline data...] } }
```

**POST /scans/:scanId/retry** — Retry a failed scan

**GET /scans/:id/diff** — Scan-to-scan diff (✅ DONE)

### Schedules

**GET /scan-schedules** — List schedules

**POST /scan-schedules** — Create schedule
```json
{ "cronExpression": "0 2 * * 1", "targets": [...], "projectId": "xxx" }
```

**POST /scan-schedules/:id/toggle** — Pause/unpause

**POST /scan-schedules/run-due** — Trigger due schedules

### Imports

**POST /imports/xml** — Upload XML scan data (multipart/form-data)

**GET /imports** — List imports

**GET /imports/quality-trend** — Quality analytics over time

**GET /imports/quality-digest** — Quality summary

**GET /imports.csv** — CSV export

**GET /imports/:id** — Import detail

**GET /imports/:id/reject-artifact** — Download reject artifact

**GET /imports/:id/annotations** — Get annotations

**POST /imports/:id/annotations** — Add annotation
```json
{ "text": "Reviewed — false positive in scanner config" }
```

**GET /imports/:id/diff** — Import-to-import diff (✅ DONE)

### Import Policies

**GET /import-policies** — List policies

**POST /import-policies** — Create policy
```json
{ "name": "Nessus Prod", "qualityMode": "strict", "description": "..." }
```

### Assets

**GET /assets** — List assets
```
?badges=true&projectId=xxx&page=1&limit=50
```
When `badges=true`, includes `badge` field: `new`|`new_this_week`|`changed`|`null`

**GET /assets/:assetId** — Asset detail

**POST /assets/backfill-identity** — Recalculate identity keys

**GET /assets/:id/annotations** — Get annotations

**POST /assets/:id/annotations** — Add annotation

### Vulnerabilities

**GET /vulns** — List vulnerabilities
```
?hasExploit=true&severity=critical&remediationStatus=open&page=1&limit=50
// Future filters: ?isKev=true&slaBreached=true&sortBy=composite
```

**PATCH /vulns/:id** — Update remediation
```json
{ "assignedTo": "jane@example.com", "dueDate": "2026-04-01", "remediationStatus": "in_progress" }
```

**POST /vulns/bulk-update** — Bulk update (PARTIAL — extends in US-7.3.04)
```json
// Current: { "ids": ["id1", "id2"], "remediationStatus": "resolved" }
// After US-7.3.04: + assignedTo, dueDate, notes
```

**GET /vulns/:cve/blast-radius** — Blast radius for specific CVE
```json
// Response
{ "cve": "CVE-2024-1234", "affectedHosts": 12, "assets": [...] }
```

**GET /vulns/stats/exploitability** — Exploitability statistics
```json
// Response
{ "total": 720, "withExploit": 89, "theoretical": 631, "bySeverity": {...} }
```

### Network

**GET /network** — Network topology graph data

**POST /network/attack-path** — Simulate attack path
```json
// Request
{ "entryAssetId": "xxx", "targetAssetId": "yyy" }
// Response
{ "path": [{ "assetId": "...", "hops": [...], "vulns": [...] }], "totalHops": 3 }
```

### Search

**GET /search** — Global Cmd+K search
```
?q=192.168.1&type=all
```
Returns mixed results: assets, vulns, scans, projects.

### Dashboard

**GET /dashboard/summary** — Project summary stats
```json
{ "totalAssets": 480, "totalVulns": 720, "criticalVulns": 45, "openRemediations": 312, ... }
```

### Reports

**GET /reports** — List generated reports

**GET /reports/imports/:id.pdf** — Generate import PDF

**GET /reports/scans/:id.pdf** — Generate scan PDF

### Vuln Enrichment

**POST /imports/:id/vuln-enrich** — Trigger vulnerability enrichment

## Planned Endpoints (Sprint 0+)

### Auth & Users (INFRA-01, US-8.08)

**POST /auth/login** — Session login
```json
{ "email": "admin@example.com", "password": "xxx" }
// Response: Set-Cookie + { "user": {...}, "token": "..." }
```

**POST /auth/logout** — Destroy session

**GET /users/me** — Current user profile

**GET /users** — List users (owner/admin only)

**POST /users** — Create user (owner only)

### API Keys (US-8.08)

**POST /api-keys** — Generate API key
```json
{ "name": "CI/CD Pipeline", "projectId": "xxx", "permissions": ["read:vulns", "read:assets"] }
// Response: 201 { "key": "armadillo_xxx...xxx", "id": "...", "keyPrefix": "armadill" }
// NOTE: Raw key shown ONCE. Only hash stored.
```

**GET /api-keys** — List API keys (metadata only, never the raw key)

**DELETE /api-keys/:id** — Revoke key

### Intelligence (US-INT.01–03)

**GET /vulns** — Extended with EPSS/KEV/composite fields
```json
// Additional response fields per vuln:
{ "epssScore": 0.89, "epssPercentile": 0.97, "isInCisaKev": true, "kevDueDate": "2026-04-01", "compositeScore": 87.5 }
```

**GET /vulns/stats/epss** — EPSS distribution stats

**GET /projects/:id/scoring-weights** — Read composite score weights
```json
{ "cvss": 0.25, "epss": 0.30, "kev": 0.20, "blast": 0.15, "exposure": 0.10 }
```

**PUT /projects/:id/scoring-weights** — Update weights (triggers recalculation)

### AI (US-AI.01–05)

**POST /ai/remediation** — Generate remediation guidance
```json
// Request
{ "vulnId": "xxx", "cveId": "CVE-2024-1234", "assetContext": { "os": "Ubuntu 22.04", "services": ["ssh", "nginx"] } }
// Response
{ "guidance": "Step-by-step...", "cached": true, "model": "anthropic/claude-sonnet-4" }
```

**POST /ai/search** — Natural language search (US-AI.02)
```json
{ "query": "show me all critical Windows vulns with public exploits" }
// Response: structured query result
```

**POST /ai/digest** — Generate scan digest (US-AI.03)

**POST /ai/classify-asset** — Auto-classify asset type/role (US-AI.05)

### Scan Credentials (INFRA-08)

**POST /scan-credentials** — Store encrypted credential
```json
{ "name": "Prod SSH", "credentialType": "ssh_password", "payload": { "username": "scanner", "password": "xxx" } }
// Payload encrypted at rest. NEVER returned in GET.
```

**GET /scan-credentials** — List credentials (metadata only)

**DELETE /scan-credentials/:id** — Delete credential

### Evidence (US-7.3.02)

**POST /evidence** — Upload evidence file (multipart/form-data)
```json
// Response: 201
{ "id": "xxx", "s3Key": "armadillo-evidence/projId/vulnId/screenshot.png", "sha256Hash": "abc..." }
```

**GET /evidence/:id/download** — Download evidence file (presigned URL)
