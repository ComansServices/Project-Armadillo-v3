## Document Authority Hierarchy

When two documents disagree, the HIGHER-ranked document wins:

```
RANK 1 (highest) ─── sprint-status.md
                         Live task tracker. Current status of every item.
                         Updated after every task completion.

RANK 2 ──────────── BUILD_PLAN.md
                         Task breakdowns, acceptance criteria, files to touch.
                         Updated at sprint start.

RANK 3 ──────────── Sprint_Plan_Sequencing.md
                         Sprint assignments, effort estimates, dependency chains.
                         Updated at planning time.

RANK 4 ──────────── Project_Armadillo_v3_Agent_Specification.md
                         Original spec. 26 sections. Prisma models, API contracts,
                         architecture. Updated at major version bumps only.

RANK 5 (lowest) ─── All other docs (ARCHITECTURE, RUNBOOK, DECISIONS, etc.)
                         Reference material. Derived from Rank 1–4.
```

### Conflict Resolution Examples

| Conflict | Winner | Reason |
|---|---|---|
| sprint-status says INFRA-01 is ✅ done, BUILD_PLAN says it's not started | sprint-status | Rank 1 > Rank 2 |
| BUILD_PLAN says US-AI.01 effort is 5d, Sprint Plan says 3d | BUILD_PLAN | Rank 2 > Rank 3 |
| BUILD_PLAN says use `fastify-type-provider-zod`, spec doesn't mention it | BUILD_PLAN | More recent, more specific |
| Spec says 63 user stories, BUILD_PLAN says 69 items | Both correct | 63 = per-section count, 69 = unique deliverables (see reconciliation) |

## Pre-Task Dependency Check

Before starting any task, run this check:

```
1. Open sprint-status.md
2. Find your task's row
3. Check the "Depends On" column
4. For each dependency:
   a. Is it marked [x] (done)? → Proceed
   b. Is it marked [~] (in progress)? → WAIT or coordinate
   c. Is it marked [ ] (not started)? → STOP, report blocker
5. If all deps are [x] → Start work
```

---

## Post-Task Update Protocol

After completing a task:

```
1. Update sprint-status.md:
   - Change [ ] to [x] for your task
   - Add completion date in the log section
   
2. Verify no downstream tasks were unblocked:
   - Search sprint-status.md for your task ID in "Depends On" columns
   - If found, note which tasks are now unblocked
   
3. Commit with standard message:
   git commit -m "[STORY-ID] description"
   git push origin HEAD

4. End with checkpoint:
   ## Checkpoint
   1. Changed: [files]
   2. Tests: [results]
   3. Blockers: [none or description]
   4. Unblocked: [downstream tasks now ready]
```
